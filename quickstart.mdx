Choose how you want to run zot: with Docker for the fastest start, or with a binary if you prefer not to use containers.

<Tabs>
  <Tab title="Docker">
    <Steps>
      <Step title="Pull the image">
        Pull the official zot image from GitHub Container Registry:

        ```bash
        docker pull ghcr.io/project-zot/zot-linux-amd64:latest
        ```
      </Step>

      <Step title="Create a config file">
        Create a minimal `config.json` in your working directory:

        ```json config.json
        {
          "distSpecVersion": "1.1.1",
          "storage": {
            "rootDirectory": "/var/lib/registry"
          },
          "http": {
            "address": "0.0.0.0",
            "port": "5000"
          },
          "log": {
            "level": "info"
          }
        }
        ```

        <Note>
          `address: "0.0.0.0"` makes the registry reachable from outside the container. For production, bind to a specific interface and enable TLS.
        </Note>
      </Step>

      <Step title="Start the registry">
        Mount the config file and a data volume, then start the container:

        ```bash
        docker run -d \
          --name zot \
          -p 5000:5000 \
          -v $(pwd)/config.json:/etc/zot/config.json \
          -v zot-data:/var/lib/registry \
          ghcr.io/project-zot/zot-linux-amd64:latest \
          serve /etc/zot/config.json
        ```

        Verify the registry is up:

        ```bash
        curl http://localhost:5000/v2/
        ```

        Expected response:

        ```json
        {}
        ```
      </Step>

      <Step title="Push an image">
        Tag a local image and push it to the registry:

        ```bash
        docker tag ubuntu:22.04 localhost:5000/ubuntu:22.04
        docker push localhost:5000/ubuntu:22.04
        ```
      </Step>

      <Step title="Pull the image back">
        ```bash
        docker pull localhost:5000/ubuntu:22.04
        ```
      </Step>
    </Steps>
  </Tab>

  <Tab title="Binary">
    <Steps>
      <Step title="Download the binary">
        Download the latest release from [GitHub Releases](https://github.com/project-zot/zot/releases/latest).

        For Linux on amd64:

        ```bash
        curl -Lo zot https://github.com/project-zot/zot/releases/latest/download/zot-linux-amd64
        chmod +x zot
        sudo mv zot /usr/local/bin/zot
        ```

        Verify the installation:

        ```bash
        zot --version
        ```
      </Step>

      <Step title="Create a config file">
        Create a minimal `config.json`:

        ```json config.json
        {
          "distSpecVersion": "1.1.1",
          "storage": {
            "rootDirectory": "/tmp/zot"
          },
          "http": {
            "address": "127.0.0.1",
            "port": "5000"
          },
          "log": {
            "level": "info"
          }
        }
        ```

        You can validate the config before starting:

        ```bash
        zot verify config.json
        ```
      </Step>

      <Step title="Start the registry">
        ```bash
        zot serve config.json
        ```

        In another terminal, check that the registry is responding:

        ```bash
        curl http://127.0.0.1:5000/v2/
        ```
      </Step>

      <Step title="Push an image">
        ```bash
        docker tag ubuntu:22.04 localhost:5000/ubuntu:22.04
        docker push localhost:5000/ubuntu:22.04
        ```
      </Step>

      <Step title="Pull the image back">
        ```bash
        docker pull localhost:5000/ubuntu:22.04
        ```
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Verify OCI compliance

The `/v2/` endpoint is the OCI Distribution Spec health check. A `200 OK` response with an empty JSON object confirms the registry is running and conformant:

```bash
curl -s http://localhost:5000/v2/ | jq
```

```json
{}
```

## List repositories

Once you have pushed an image, you can list repositories through the standard OCI catalog endpoint:

```bash
curl -s http://localhost:5000/v2/_catalog | jq
```

```json
{
  "repositories": ["ubuntu"]
}
```

## Using zli

`zli` is the companion CLI for zot. It lets you search images, list tags, and manage API keys.

<Steps>
  <Step title="Download zli">
    ```bash
    curl -Lo zli https://github.com/project-zot/zot/releases/latest/download/zli-linux-amd64
    chmod +x zli
    sudo mv zli /usr/local/bin/zli
    ```
  </Step>

  <Step title="Add a registry config">
    ```bash
    zli config add local http://localhost:5000
    ```
  </Step>

  <Step title="List repositories">
    ```bash
    zli repo list --config local
    ```
  </Step>

  <Step title="List images">
    ```bash
    zli image list --config local
    ```
  </Step>
</Steps>

## Example: config with authentication

The minimal config above runs without authentication. To add htpasswd-based basic auth, generate a credentials file and reference it in the config:

```bash
# Install apache2-utils for htpasswd
htpasswd -cB /etc/zot/htpasswd alice
```

```json config.json
{
  "distSpecVersion": "1.1.1",
  "storage": {
    "rootDirectory": "/var/lib/registry"
  },
  "http": {
    "address": "0.0.0.0",
    "port": "5000",
    "auth": {
      "htpasswd": {
        "path": "/etc/zot/htpasswd"
      },
      "failDelay": 5
    }
  },
  "log": {
    "level": "info"
  }
}
```

Push with credentials:

```bash
docker login localhost:5000 -u alice
docker push localhost:5000/ubuntu:22.04
```
