zot is distributed as a container image, pre-built binaries for Linux, macOS, and Windows, and a Helm chart for Kubernetes. Choose the method that fits your environment.

<Tabs>
  <Tab title="Docker">
    ## Docker / container image

    Official images are published to GitHub Container Registry under the `ghcr.io/project-zot` organization.

    ### Image naming

    Images follow the pattern `ghcr.io/project-zot/zot-<os>-<arch>`. The most common variants are:

    | Image | Description |
    |-------|-------------|
    | `ghcr.io/project-zot/zot-linux-amd64` | Full build for Linux x86-64 |
    | `ghcr.io/project-zot/zot-linux-arm64` | Full build for Linux ARM64 |
    | `ghcr.io/project-zot/zot-linux-amd64-minimal` | Minimal build for Linux x86-64 (no extensions) |

    ### Pull and run

    ```bash
    docker pull ghcr.io/project-zot/zot-linux-amd64:latest
    ```

    Run with a config file and a persistent volume:

    ```bash
    docker run -d \
      --name zot \
      -p 5000:5000 \
      -v $(pwd)/config.json:/etc/zot/config.json \
      -v zot-data:/var/lib/registry \
      ghcr.io/project-zot/zot-linux-amd64:latest \
      serve /etc/zot/config.json
    ```

    ### Pin a specific version

    Replace `latest` with a release tag (e.g. `v2.1.0`):

    ```bash
    docker pull ghcr.io/project-zot/zot-linux-amd64:v2.1.0
    ```

    ### Minimal config for Docker

    ```json config.json
    {
      "distSpecVersion": "1.1.1",
      "storage": {
        "rootDirectory": "/var/lib/registry"
      },
      "http": {
        "address": "0.0.0.0",
        "port": "5000"
      },
      "log": {
        "level": "info"
      }
    }
    ```

    <Tip>
      Use `address: "0.0.0.0"` when running inside a container so the port is reachable from the host. For production, put a TLS-terminating reverse proxy in front of zot.
    </Tip>
  </Tab>

  <Tab title="Pre-built binary">
    ## Pre-built binary

    Pre-built binaries are available for Linux, macOS, and Windows on the [GitHub Releases page](https://github.com/project-zot/zot/releases).

    ### Binary variants

    Each release publishes two binary types per platform:

    | Suffix | Extensions included |
    |--------|---------------------|
    | _(none)_ e.g. `zot-linux-amd64` | Full: search, sync, metrics, lint, UI, image trust, scrub, user preferences, events |
    | `-minimal` e.g. `zot-linux-amd64-minimal` | None — core OCI registry only |

    ### Linux (amd64)

    ```bash
    # Full build
    curl -Lo zot https://github.com/project-zot/zot/releases/latest/download/zot-linux-amd64
    chmod +x zot
    sudo mv zot /usr/local/bin/zot
    ```

    ```bash
    # Minimal build
    curl -Lo zot https://github.com/project-zot/zot/releases/latest/download/zot-linux-amd64-minimal
    chmod +x zot
    sudo mv zot /usr/local/bin/zot
    ```

    ### Linux (arm64)

    ```bash
    curl -Lo zot https://github.com/project-zot/zot/releases/latest/download/zot-linux-arm64
    chmod +x zot
    sudo mv zot /usr/local/bin/zot
    ```

    ### macOS (arm64 / Apple Silicon)

    ```bash
    curl -Lo zot https://github.com/project-zot/zot/releases/latest/download/zot-darwin-arm64
    chmod +x zot
    sudo mv zot /usr/local/bin/zot
    ```

    ### Verify and run

    Check the binary works:

    ```bash
    zot --version
    ```

    Validate a config file:

    ```bash
    zot verify config.json
    ```

    Start the registry:

    ```bash
    zot serve config.json
    ```

    ### Companion tools

    The same release page provides:

    | Binary | Description |
    |--------|-------------|
    | `zli-linux-amd64` | CLI client for querying and managing a zot registry |
    | `zb-linux-amd64` | Benchmarking tool |
    | `zxp-linux-amd64` | Standalone Prometheus metrics exporter (for use with minimal builds) |

    ```bash
    curl -Lo zli https://github.com/project-zot/zot/releases/latest/download/zli-linux-amd64
    chmod +x zli
    sudo mv zli /usr/local/bin/zli
    ```

  <Tab title="Build from source">
    ## Build from source

    Building from source requires Go 1.21 or later.

    ### Clone the repository

    ```bash
    git clone https://github.com/project-zot/zot.git
    cd zot
    ```

    ### Build the full binary

    The full binary includes all extensions: `sync`, `search`, `scrub`, `metrics`, `lint`, `ui`, `mgmt`, `profile`, `userprefs`, `imagetrust`, and `events`.

    ```bash
    make binary
    ```

    The binary is written to `bin/zot-<os>-<arch>`.

    ### Build the minimal binary

    The minimal binary includes no extensions — it is a pure OCI Distribution Spec registry:

    ```bash
    make binary-minimal
    ```

    The binary is written to `bin/zot-<os>-<arch>-minimal`.

    ### Build with a custom extension set

    Set the `EXTENSIONS` variable to a comma-separated list of the extensions you want:

    ```bash
    EXTENSIONS=search,metrics make binary
    ```

    Available extension names: `debug`, `imagetrust`, `lint`, `metrics`, `mgmt`, `profile`, `scrub`, `search`, `sync`, `ui`, `userprefs`, `events`.

    ### Build the CLI (`zli`)

    ```bash
    make cli
    ```

    The binary is written to `bin/zli-<os>-<arch>`.

    ### Build for a different OS or architecture

    ```bash
    OS=linux ARCH=arm64 make binary
    ```

    ### Environment variable reference

    | Variable | Default | Description |
    |----------|---------|-------------|
    | `EXTENSIONS` | `sync,search,scrub,metrics,lint,ui,mgmt,profile,userprefs,imagetrust,events` | Comma-separated extension list |
    | `OS` | Current OS | Target operating system (`linux`, `darwin`, `windows`, `freebsd`) |
    | `ARCH` | Current arch | Target architecture (`amd64`, `arm64`) |

    <Note>
      The build sets `CGO_ENABLED=0` and uses `GOEXPERIMENT=jsonv2` by default. The resulting binary is statically linked and does not require any shared libraries at runtime.
    </Note>
  </Tab>

  <Tab title="Helm (Kubernetes)">
    ## Helm chart

    The official Helm chart is published to the `project-zot` chart repository.

    ### Add the Helm repository

    ```bash
    helm repo add project-zot https://zotregistry.dev/helm-charts
    helm repo update
    ```

    ### Install with default values

    ```bash
    helm install zot project-zot/zot \
      --namespace registry \
      --create-namespace
    ```

    ### Install with a custom config

    Create a `values.yaml` file that overrides the registry config:

    ```yaml values.yaml
    configFiles:
      config.json: |
        {
          "distSpecVersion": "1.1.1",
          "storage": {
            "rootDirectory": "/var/lib/registry"
          },
          "http": {
            "address": "0.0.0.0",
            "port": "5000"
          },
          "log": {
            "level": "info"
          }
        }
    ```

    ```bash
    helm install zot project-zot/zot \
      --namespace registry \
      --create-namespace \
      -f values.yaml
    ```

    ### Upgrade

    ```bash
    helm upgrade zot project-zot/zot \
      --namespace registry \
      -f values.yaml
    ```

    ### Uninstall

    ```bash
    helm uninstall zot --namespace registry
    ```

    <Tip>
      For production Kubernetes deployments, see the [Kubernetes guide](/guides/kubernetes) for persistent volume configuration, ingress TLS termination, and horizontal scaling with a shared S3 backend.
    </Tip>
  </Tab>

## Minimal config reference

Every zot instance requires a configuration file with at least the following four fields:

```json config.json
{
  "distSpecVersion": "1.1.1",
  "storage": {
    "rootDirectory": "/var/lib/registry"
  },
  "http": {
    "address": "127.0.0.1",
    "port": "5000"
  },
  "log": {
    "level": "info"
  }
}
```

| Field | Description |
|-------|-------------|
| `distSpecVersion` | OCI Distribution Spec version the server advertises. Use `"1.1.1"`. |
| `storage.rootDirectory` | Directory where OCI image data is stored. |
| `http.address` | IP address to listen on. Use `"0.0.0.0"` to listen on all interfaces. |
| `http.port` | TCP port to listen on. |
| `log.level` | Log verbosity: `"debug"`, `"info"`, `"warn"`, `"error"`, `"fatal"`, `"panic"`. |

Validate any config file before starting the registry:

```bash
zot verify config.json
```
