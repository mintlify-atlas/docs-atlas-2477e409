The search extension exposes a GraphQL API at `/v2/_zot/ext/search` for querying images, repositories, and vulnerability data. When combined with Trivy configuration, it also runs periodic CVE scans and serves vulnerability results through the same API.

## Enabling search

The search extension is compiled in using the `search` build tag. Enable it in the configuration file:

```json
{
  "extensions": {
    "search": {
      "enable": true
    }
  }
}
```

## Enabling CVE scanning

CVE scanning requires a Trivy database. Add the `cve` block to configure the update interval and (optionally) custom database repositories:

```json
{
  "extensions": {
    "search": {
      "enable": true,
      "cve": {
        "updateInterval": "24h",
        "trivy": {
          "dbRepository": "ghcr.io/aquasecurity/trivy-db",
          "javaDBRepository": "ghcr.io/aquasecurity/trivy-java-db"
        }
      }
    }
  }
}
```

<ParamField path="extensions.search.enable" type="boolean" required>
  Set to `true` to activate the GraphQL search endpoint.
</ParamField>

<ParamField path="extensions.search.cve" type="object">
  CVE scanning configuration. If omitted, the search endpoint works but vulnerability queries return no data.
</ParamField>

<ParamField path="extensions.search.cve.updateInterval" type="string">
  How often to refresh the Trivy vulnerability database. Must be at least `"2h"`. If not specified, defaults to `"2h"`. Accepts Go duration strings.
</ParamField>

<ParamField path="extensions.search.cve.trivy.dbRepository" type="string" default="ghcr.io/aquasecurity/trivy-db">
  OCI repository from which to download the Trivy vulnerability database.
</ParamField>

<ParamField path="extensions.search.cve.trivy.javaDBRepository" type="string" default="ghcr.io/aquasecurity/trivy-java-db">
  OCI repository from which to download the Trivy Java advisory database.
</ParamField>

## GraphQL endpoint

All queries go to `POST /v2/_zot/ext/search` (or `GET` for simple queries). The endpoint accepts standard GraphQL request bodies:

```bash
curl -X POST http://localhost:5000/v2/_zot/ext/search \
  -H "Content-Type: application/json" \
  -d '{"query": "{ ImageList(repo: \"ubuntu\") { Results { Tag Digest } } }"}'
```

## Query examples

<CodeGroup>

```graphql List images in a repository
{
  ImageList(repo: "ubuntu") {
    Results {
      Tag
      Digest
      LastUpdated
      Size
      IsSigned
      Vulnerabilities {
        MaxSeverity
        Count
        CriticalCount
        HighCount
        MediumCount
        LowCount
      }
    }
  }
}
```

```graphql List CVEs for an image
{
  CVEListForImage(image: "ubuntu:22.04") {
    Tag
    CVEList {
      Id
      Title
      Description
      Severity
      PackageList {
        Name
        InstalledVersion
        FixedVersion
      }
    }
    Summary {
      MaxSeverity
      Count
      CriticalCount
      HighCount
    }
  }
}
```

```graphql Images where a CVE is fixed
{
  ImageListWithCVEFixed(id: "CVE-2023-12345", image: "ubuntu") {
    Results {
      Tag
      Digest
      LastUpdated
    }
  }
}
```

```graphql Images affected by a CVE
{
  ImageListForCVE(id: "CVE-2023-12345") {
    Results {
      RepoName
      Tag
      Digest
    }
  }
}
```

```graphql Global search
{
  GlobalSearch(query: "ubuntu") {
    Images {
      RepoName
      Tag
      Digest
      LastUpdated
    }
    Repos {
      Name
      LastUpdated
      StarCount
    }
  }
}
```

```graphql CVE diff between two images
{
  CVEDiffListForImages(
    minuend: { Repo: "ubuntu", Tag: "22.04" }
    subtrahend: { Repo: "ubuntu", Tag: "22.10" }
  ) {
    CVEList {
      Id
      Severity
      Title
    }
    Summary {
      Count
      CriticalCount
    }
  }
}
```

</CodeGroup>

## Query reference

The following queries are available on the `Query` type:

| Query | Description |
|-------|-------------|
| `ImageList(repo)` | All images in a given repository |
| `CVEListForImage(image)` | CVEs affecting a specific image tag or digest |
| `CVEDiffListForImages(minuend, subtrahend)` | CVEs in `minuend` that are not present in `subtrahend` |
| `ImageListForCVE(id)` | Images that are vulnerable to a specific CVE |
| `ImageListWithCVEFixed(id, image)` | Image tags within a repository where a CVE is fixed |
| `ImageListForDigest(id)` | Images containing a specific layer or manifest digest |
| `GlobalSearch(query)` | Search across repos, images, and layers by name |
| `RepoListWithNewestImage` | Repositories sorted by most recently updated image |
| `ExpandedRepoInfo(repo)` | Full details about a repository and all its images |
| `DerivedImageList(image)` | Images that use the given image as a base layer |
| `BaseImageList(image)` | Base images that the given image depends on |
| `Image(image)` | Full details about a specific image |
| `Referrers(repo, digest)` | OCI referrers attached to a specific manifest digest |
| `StarredRepos` | Repositories starred by the current user |
| `BookmarkedRepos` | Repositories bookmarked by the current user |

## Pagination

Most list queries accept a `requestedPage` argument for pagination:

```graphql
{
  ImageList(
    repo: "ubuntu"
    requestedPage: { limit: 10, offset: 0, sortBy: UPDATE_TIME }
  ) {
    Page {
      TotalCount
      ItemCount
    }
    Results {
      Tag
      LastUpdated
    }
  }
}
```

Available sort criteria: `RELEVANCE`, `UPDATE_TIME`, `ALPHABETIC_ASC`, `ALPHABETIC_DSC`, `SEVERITY`, `STARS`, `DOWNLOADS`.

## Filtering

Queries that return image or repository lists accept a `filter` argument:

```graphql
{
  ImageListForCVE(
    id: "CVE-2023-12345"
    filter: {
      Os: ["linux"]
      Arch: ["amd64"]
      HasToBeSigned: true
    }
  ) {
    Results {
      RepoName
      Tag
    }
  }
}
```

Filter fields: `Os`, `Arch`, `HasToBeSigned`, `IsBookmarked`, `IsStarred`.

## CVE severity levels

CVE severity maps to CVSS v3 score bands:

| Severity | CVSS v3 range |
|----------|--------------|
| `CRITICAL` | 9.0–10.0 |
| `HIGH` | 7.0–8.9 |
| `MEDIUM` | 4.0–6.9 |
| `LOW` | 0.1–3.9 |
| `UNKNOWN` | No score available |

<Warning>
The first CVE database download occurs after zot starts. Vulnerability data will not be available until the initial download completes, which can take several minutes depending on network speed.
</Warning>
