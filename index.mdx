zot is a scale-out, production-ready container image registry that stores images in [OCI image format](https://github.com/opencontainers/image-spec) and communicates using the [OCI Distribution Specification](https://github.com/opencontainers/distribution-spec) on-the-wire. No vendor lock-in, no proprietary extensions — just OCI, all the way down.

## Why zot?

zot is the only fully OCI-native registry — every image is stored on disk in the OCI Image Layout format, so your data is always portable and never locked in to a proprietary format.

## Get started

<Steps>
  <Step title="Install zot">
    Download a pre-built binary or pull the container image:
    ```bash
    docker pull ghcr.io/project-zot/zot-linux-amd64:latest
    ```
  </Step>
  <Step title="Create a minimal config">
    Create `config.json` with basic settings:
    ```json config.json
    {
      "distSpecVersion": "1.1.1",
      "storage": { "rootDirectory": "/var/lib/registry" },
      "http": { "address": "0.0.0.0", "port": "5000" },
      "log": { "level": "info" }
    }
    ```
  </Step>
  <Step title="Start the registry">
    ```bash
    zot serve config.json
    ```
  </Step>
  <Step title="Push your first image">
    ```bash
    docker tag myimage localhost:5000/myimage:latest
    docker push localhost:5000/myimage:latest
    ```
  </Step>
</Steps>
