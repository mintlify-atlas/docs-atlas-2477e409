zot is an open source container image registry that stores images in [OCI Image Layout](https://github.com/opencontainers/image-spec) format and communicates exclusively using the [OCI Distribution Specification](https://github.com/opencontainers/distribution-spec) on the wire. No proprietary extensions, no vendor lock-in — just OCI, end to end.

## What is OCI?

The [Open Container Initiative (OCI)](https://opencontainers.org) maintains two specifications that govern how containers are stored and distributed:

- **OCI Image Spec** — defines how a container image is laid out on disk, including layers, manifests, and configuration.
- **OCI Distribution Spec** — defines the HTTP API that registries expose for pushing and pulling images.

zot is fully conformant with OCI Distribution Specification v1.1.1 and stores every image directly in the OCI Image Layout format. This means your data is always portable: you can move your storage root to any other OCI-conformant tool without conversion.

## Binary types

zot ships two binary variants:

| Variant | Description |
|---------|-------------|
| **Full** (`zot-linux-amd64`) | Includes all extensions: search, sync, metrics, lint, UI, image trust, scrub, user preferences, and events. |
| **Minimal** (`zot-linux-amd64-minimal`) | Core OCI registry only, no extensions. Smaller footprint for constrained environments. |

The full binary is built with Go build tags corresponding to each extension (`search`, `sync`, `metrics`, `lint`, `ui`, `imagetrust`, `scrub`, `userprefs`, `events`). You can also build a custom binary with only the extensions you need.

## Configuration overview

zot is configured with a single JSON or YAML file. Every configuration has four required top-level sections:

```json
{
  "distSpecVersion": "1.1.1",
  "storage": {
    "rootDirectory": "/var/lib/registry"
  },
  "http": {
    "address": "0.0.0.0",
    "port": "5000"
  },
  "log": {
    "level": "info"
  }
}
```

Extensions are enabled under the optional `extensions` key. Authentication is configured under `http.auth`, and access control policies go under `http.accessControl`.

## Tools

zot ships with companion tools in the same release:

- **`zli`** — a CLI client for interacting with a zot registry (searching images, managing repositories, handling API keys).
- **`zb`** — a benchmarking tool for load-testing a registry.
- **`zxp`** — a standalone Prometheus metrics exporter for use with the minimal build.
